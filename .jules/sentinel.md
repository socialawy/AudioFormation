## 2024-03-20 - [Security Learning: Path Traversal Bypass via os.path.abspath]
**Vulnerability:** The function `validate_path_within` in `src/audioformation/utils/security.py` uses `os.path.abspath` instead of `Path.resolve()` for path validation. `os.path.abspath` only normalizes path strings and does not resolve symlinks. This allows a path traversal bypass where an attacker could provide a symlink that resolves outside the intended root directory, and `validate_path_within` would incorrectly deem it safe.
**Learning:** Pure string manipulation for path validation is insufficient when dealing with a file system that supports symlinks. Symbolic links can point to arbitrary locations on the disk, and checking the string representation of a path does not prevent access to those locations.
**Prevention:** Always use `Path.resolve().is_relative_to(root.resolve())` when validating if a path falls within a specific directory. This ensures symlinks are fully evaluated against the actual file system structure before the check is performed. Additionally, handle potential exceptions (e.g., `RuntimeError` for infinite symlink loops) during resolution to fail securely.

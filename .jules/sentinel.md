## $(date +%Y-%m-%d) - [CRITICAL] Fix Exposed Configuration Files in Static Mount
**Vulnerability:** The `/projects` static file mount used FastAPI's default `StaticFiles`, which inadvertently exposed all files inside a project directory including sensitive files like `00_CONFIG/engines.json` and `.env` to any unauthenticated web user.
**Learning:** Default static file servers map directly to the filesystem and do not distinguish between public assets (like generated audio/mixes) and backend configuration data stored alongside them. When creating directory structures that mix public and private data under one root, it's trivial to accidentally leak credentials via the static mount.
**Prevention:** Override the `lookup_path` method of `StaticFiles` (e.g., creating a `SafeStaticFiles` class) to explicitly intercept and block access to sensitive directories (e.g., `00_config`), hidden files (`.*`), and sensitive extensions (`.env`, `.yml`), and use case-insensitive matching to prevent evasion on Windows/macOS filesystems.

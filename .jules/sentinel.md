## 2025-02-13 - [Path Traversal Bypass via Symlinks]
**Vulnerability:** The function `validate_path_within` in `src/audioformation/utils/security.py` used `os.path.abspath` to ensure a file path was within a specific root directory. `os.path.abspath` handles path resolution via string manipulation (normalizing `..` and `.`) but does not resolve filesystem symlinks. Consequently, an attacker could bypass path traversal protection by creating a symlink inside the safe directory that points to an external file.
**Learning:** Pure string manipulation techniques like `os.path.abspath` or `os.path.normpath` are insufficient to prevent path traversal when symlinks are involved. Without querying the filesystem to resolve actual targets, malicious symlinks will be mistakenly treated as valid internal paths.
**Prevention:** To prevent path traversal bypasses, path validation logic must access the filesystem and always use `Path.resolve().is_relative_to()` instead of string-based manipulation to securely handle symlinks. Exceptions `(TypeError, ValueError, RuntimeError)` should be caught to gracefully handle potential infinite symlink loops and malformed paths to ensure secure failure.

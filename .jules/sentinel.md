## 2024-05-18 - Path Traversal Bypass via Symlinks
**Vulnerability:** The `validate_path_within` function previously used string manipulation (`os.path.abspath`) to determine if a path was within an allowed directory. This approach is vulnerable to symlink-based path traversal because `os.path.abspath` normalizes strings (`..`) but does not interact with the filesystem to resolve symlinks. Consequently, a symlink placed inside the allowed directory could point to an external file (e.g., `/etc/passwd`), and string-based validation would incorrectly permit access to it.
**Learning:** String-based path validation is fundamentally flawed when dealing with file systems that support symlinks. Validation logic must interact with the actual filesystem to resolve the true canonical path before asserting directory boundaries.
**Prevention:** To prevent symlink bypasses, path validation logic must always use `Path.resolve().is_relative_to()` to evaluate paths securely and resolve any underlying symlinks. Additionally, gracefully catch `(TypeError, ValueError, RuntimeError)` to prevent infinite symlink loops or malformed paths from crashing the application, enforcing a secure fail-closed pattern.

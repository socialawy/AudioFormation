## 2025-02-17 - [Path Traversal bypass due to string-based checks]
**Vulnerability:** The application was using purely string-based manipulation (`os.path.abspath`) in `validate_path_within` to verify that an untrusted path resolves inside a secure root directory. This allows a path traversal bypass because `os.path.abspath` does not resolve symlinks on disk, meaning a crafted symlink inside the root directory could point anywhere on the system (like `/etc/passwd`) and be incorrectly deemed safe because its string representation appears within the root.
**Learning:** `os.path.abspath` normalizes `.` and `..` via text manipulation, making it inadequate for filesystem security checks when symlinks might be involved. Secure path validation must always interact with the actual filesystem to resolve the ultimate target location.
**Prevention:** Always use `Path.resolve().is_relative_to(root.resolve())` in path validation logic instead of `os.path.abspath` to securely handle symlinks and enforce directory containment.
